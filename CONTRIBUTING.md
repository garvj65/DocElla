# Contributing

`main` should remain deployable. After the initial scaffold, use short-lived branches for new work.

Codex-created branches should follow:

```text
agent/tXX-short-description
```

Examples:

```text
agent/t02-document-definitions
agent/t03-backend-foundation
```

Use Conventional Commit-style messages. Keep commits focused and recoverable.

Do not force-push `main`. Open draft pull requests for incomplete work. Run repository gates before pushing.

Do not commit secrets, PDFs containing real personal information, or raw extracted customer data.
